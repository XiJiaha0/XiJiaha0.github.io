多线程

  • 进程

    内存中执行的程序叫进程，在内存中独立运行(隔离)的程序空间，因此不同进程间数据难以共享

    进程不会相互影响，但会受内部线程影响

    内存可以上锁以禁止其他进程访问 8核=模拟轮流依次处理

     

  • 线程

    执行单元(如main)，一个进程至少存在一个线程，同进程下的多个线程可以共享进程的资源

    均在同一地址空间进行，线程间通信无须额外机制；启动终端消亡消耗资源极少

  • 并发

    采用时间片轮询执行多个线程，一个时间段内线程运行时其他线程处于就绪状态

    main方法作为应用入口，也是应用主线程

    JVM执行main方法会进入栈内存，并为其开辟一条通向CPU的执行路径

线程并发协作

生产者(生成数据模块)=====》》缓冲区(----生产者置入数据，消费者取出数据----)=====》》消费者(处理数据模块)

  • 缓冲区

    有了缓冲区就实现了生产者与消费者的，线程并发协作；两者互不干涉互不影响；

    解决了忙闲不均的问题； 生产数据慢，缓冲区的数据不影响消费者处理；处理数据慢不影响生产者生成；

  • 创建缓冲区

    xxxxxxxxxx
    class 资源{
    private int id;
    public 资源(int id){this.id=id;}
    }
    class 资源缓冲区{
    //定义存放定义资源数组，长度为10
    private 资源[] productSet=new 资源[10];
    //定义索引
    private int index;
    //存放 与取出操作 进行线程同步 串行执行
    public synchronized void push(资源 p){
       while(this.index==productSet.length){
           try {this.wait(1000);} //停滞且释放对象锁 允许pop进入同步块执行
           catch (Exception e) {throw new RuntimeException(e);}}
       this.productSet[this.index]=p; //资源置入
       this.index++;
       this.notify();//通知到货 唤醒处于等待状态中的一个线程
    }//wait(停滞并交给具备相同锁的线程执行)和notify必须在synchronized块内调用
    //取出
    public synchronized 资源 pop(){
       while(this.index==0){
           try {this.wait(1000);} //停滞且释放对象锁 允许pop进入同步块执行
           catch (Exception e) {throw new RuntimeException(e);}}
       this.notify();//通知进货 唤醒处于等待状态中的一个线程
       this.index--;
       return this.productSet[this.index];
    }
    }
    class 生产者 extends Thread{
    private 资源缓冲区 rs;
    public 生产者(资源缓冲区 rs){this.rs=rs;}
    @Override
    public void run() {
       for (int i = 0; i < 10; i++) {
           System.out.println(i+"生产资源");
           资源 g=new 资源(i);
           this.rs.push(g);
       }
    }
    }
    class 消费者 extends Thread{
    private 资源缓冲区 rs;
    public 消费者(资源缓冲区 rs){this.rs=rs;}
    @Override
    public void run() {
       for (int i = 0; i < 10; i++) {
           System.out.println(i+"消费资源");
           this.rs.pop();
       }
    }
    }
    public class BufferDistrict {
    public static void main(String[] args) {
       资源缓冲区 rd=new 资源缓冲区();
       new 生产者(rd).start();
       new 消费者(rd).start();
    }
    }

总结

银行系统

营业厅进程，具备金库、建筑 独立内存空间；

厅内包含多服务窗口线程，共享营业厅现金柜、数据库、叫号机资源

开门营业，经理主线程启动系统，随后多线程并发运行。

线程创建与资源继承

常规窗口继承银行统一窗口Thread类

VIP窗口服务接口Runnable+组合模块

线程生命周期

柜员下班按下终止按钮，停止接收新用户，并处理好当下手头任务；终止线程 提供Boolean型终止变量

紧急业务窗口需要让出资源 yield()

VIP客户需要两个窗口联合处理业务 join()e.g. A窗口等待B窗口完成验资

对公业务窗口需要保持最高优先级setPriority() 优先分配

保洁守护线程setDaemon()随用户线程的终止而自动结束

同步锁

现金柜台操作保险箱对象锁synchronized(this)：柜员张三🔒自己柜台，李四仍可操作相邻柜台

金库大门开关类锁synchronized(Bank.class)：操作人员🔒维护银行系统金库，其余人员无法操作

外汇交易区域自定义锁synchronized(foreignLock)：只🔒特定区域，如对象.方法

全局广播系统危险字串锁synchronized("Alert") ：🔒全行(JVM)操作暂停

死锁

会计A：先🔒账本A，再申请账本B

会计B：先🔒账本B，再申请账本A

关键机制

sleep()与wait() 前者锁定操作台但不释放资源：午休；后者等待授权，释放操作台供其他人使用
